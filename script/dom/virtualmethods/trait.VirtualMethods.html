Trait to allow DOM nodes to opt-in to overriding (or adding to) common behaviours. Replicates the effect of C++ virtual methods.

Required Methods

fn super_type<'a>(&'a self) -> Option<&'a VirtualMethods:>

Returns self as the superclass of the implementation for this trait, if any.

Provided Methods

fn after_set_attr(&self, name: DOMString, value: DOMString)

Called when changing or adding attributes, after the attribute's value has been updated.

fn before_remove_attr(&self, name: DOMString, value: DOMString)

Called when changing or removing attributes, before any modification has taken place.

fn parse_plain_attribute(&self, name: &str, value: DOMString) -> AttrValue

Returns the right AttrValue variant for the attribute with name name on this element.

fn bind_to_tree(&self, tree_in_doc: bool)

Called when a Node is appended to a tree, where 'tree_in_doc' indicates whether the tree is part of a Document.

fn unbind_from_tree(&self, tree_in_doc: bool)

Called when a Node is removed from a tree, where 'tree_in_doc' indicates whether the tree is part of a Document.

fn child_inserted(&self, child: &JSRef<Node>)

Called on the parent when a node is added to its child list.

fn handle_event(&self, event: &JSRef<Event>)

Called during event dispatch after the bubbling phase completes.
