Enum script::layout_interface::Msg[src]

pub enum Msg {
    AddStylesheetMsg(Stylesheet),
    ReflowMsg(Box<Reflow>),
    QueryMsg(LayoutQuery),
    ReapLayoutDataMsg(LayoutDataRef),
    PrepareToExitMsg(Sender<()>),
    ExitNowMsg,
}

Asynchronous messages that script can send to layout.

FIXME(pcwalton): I think this should probably be merged with LayoutQuery below.

Variants

AddStylesheetMsg

Adds the given stylesheet to the document.

ReflowMsg

Requests a reflow.

QueryMsg

Performs a synchronous layout request.

FIXME(pcwalton): As noted below, this isn't very type safe.

ReapLayoutDataMsg

Destroys layout data associated with a DOM node.

TODO(pcwalton): Maybe think about batching to avoid message traffic.

PrepareToExitMsg

Requests that the layout task enter a quiescent state in which no more messages are accepted except ExitMsg. A response message will be sent on the supplied channel when this happens.

ExitNowMsg

Requests that the layout task immediately shut down. There must be no more nodes left after this, or layout will crash.
